1 প্রোগ্রামিং প্যারাডাইম

প্রোগ্রামিং প্যারাডাইম কী?
প্রোগ্রামিং প্যারাডাইম হলো প্রোগ্রাম লেখার একটি নির্দিষ্ট পদ্ধতি বা শৈলী। এটি একটি কাঠামো যা নির্ধারণ করে কীভাবে প্রোগ্রাম ডিজাইন ও লেখা হবে।
প্যারাডাইমবিবরণউদাহরণ

4 ক্লাস (Class)

ক্লাস কী?
ক্লাস হলো অবজেক্ট তৈরির একটি ব্লুপ্রিন্ট বা টেমপ্লেট। এটি নির্ধারণ করে একটি অবজেক্টের বৈশিষ্ট্য (attributes) এবং আচরণ (methods) কী হবে।
📝 সহজ ভাষায়

ক্লাস হলো একটি ফর্মা। যেমন আমরা যখন নতুন আইডি কার্ড বানাই, তখন একটি ফর্মা থাকে। সেই ফর্মায় নাম, রোল, ছবি ইত্যাদি থাকে। ঠিক তেমনি ক্লাস হলো একটি ফর্মা।

// Java-এ ক্লাস তৈরি public class Student { // অ্যাট্রিবিউট String name; int roll; String dept; // কনস্ট্রাক্টর public Student(String name, int roll, String dept) { this.name = name; this.roll = roll; this.dept = dept; } // মেথড public void study() { System.out.println(name + " পড়াশোনা করছে"); } public void showInfo() { System.out.println("নাম: " + name + ", রোল: " + roll + ", বিভাগ: " + dept); } public static void main(String[] args) { Student s1 = new Student("রহিম", 101, "CST"); s1.showInfo(); s1.study(); } } নাম: রহিম, রোল: 101, বিভাগ: CST\nরহিম পড়াশোনা করছে

5 অবজেক্ট (Object)

অবজেক্ট কী?
অবজেক্ট হলো ক্লাসের একটি নির্দিষ্ট উদাহরণ (instance)। ক্লাস হলো টেমপ্লেট এবং অবজেক্ট হলো সেই টেমপ্লেট থেকে তৈরি প্রকৃত জিনিস।
// অবজেক্ট তৈরি Student student1 = new Student("রহিম", 101, "CST"); Student student2 = new Student("করিম", 102, "CST"); System.out.println(student1.name); // রহিম System.out.println(student2.name); // করিম student1.showInfo(); রহিম\nকরিম\nনাম: রহিম, রোল: 101, বিভাগ: CST

6 কনস্ট্রাক্টর (Constructor)

কনস্ট্রাক্টর কী?
কনস্ট্রাক্টর হলো একটি বিশেষ মেথড যা অবজেক্ট তৈরির সময় স্বয়ংক্রিয়ভাবে কল হয়। এটি অবজেক্টের প্রাথমিক অবস্থা নির্ধারণ করে।
বৈশিষ্ট্যবিবরণ
নামPython-এ __init__, Java/C++-এ ক্লাসের নামের সমান
কলঅবজেক্ট তৈরির সময় স্বয়ংক্রিয়ভাবে কল হয়
রিটার্নকোনো ভ্যালু রিটার্ন করে না
উদ্দেশ্যঅবজেক্টের প্রাথমিক মান সেট করা
// Java-এ কনস্ট্রাক্টর public class Person { String name; int age; // কনস্ট্রাক্টর public Person(String name, int age) { this.name = name; this.age = age; System.out.println(name + " অবজেক্ট তৈরি হয়েছে!"); } // ফাইনালাইজার (ডিস্ট্রাক্টরের মতো) protected void finalize() { System.out.println(name + " অবজেক্ট ধ্বংস হয়েছে!"); } public static void main(String[] args) { Person p1 = new Person("রহিম", 25); p1 = null; System.gc(); // Garbage Collector কল } } রহিম অবজেক্ট তৈরি হয়েছে!\nরহিম অবজেক্ট ধ্বংস হয়েছে!

🎤 OOP ইন্টারভিউ প্রশ্ন

OOP কী?
OOP হলো অবজেক্ট ওরিয়েন্টেড প্রোগ্রামিং। এটি একটি প্রোগ্রামিং প্যারাডাইম যেখানে ক্লাস ও অবজেক্ট ব্যবহার করে প্রোগ্রাম লেখা হয়। OOP-এর চারটি মূল স্তম্ভ: এনক্যাপসুলেশন, ইনহেরিট্যান্স, পলিমরফিজম, অ্যাবস্ট্রাকশন।
ক্লাস ও অবজেক্টের মধ্যে পার্থক্য কী?
ক্লাস হলো অবজেক্ট তৈরির ব্লুপ্রিন্ট। অবজেক্ট হলো ক্লাসের নির্দিষ্ট উদাহরণ (instance)। ক্লাস হলো ধারণা, অবজেক্ট হলো প্রকৃত জিনিস।
এনক্যাপসুলেশন ও অ্যাবস্ট্রাকশনের পার্থক্য কী?
এনক্যাপসুলেশন ডেটা সুরক্ষিত রাখে (Data Hiding), অ্যাবস্ট্রাকশন জটিলতা লুকায় (Implementation Hiding)। এনক্যাপসুলেশন প্রাইভেট মডিফায়ার দিয়ে, অ্যাবস্ট্রাকশন অ্যাবস্ট্রাক্ট ক্লাস দিয়ে করা হয়।
Method Overloading ও Overriding-এর পার্থক্য কী?
Overloading হলো একই ক্লাসে একই নামে একাধিক মেথড (আর্গুমেন্ট ভিন্ন)। Overriding হলো চাইল্ড ক্লাসে প্যারেন্টের মেথড পুনঃসংজ্ঞায়িত করা। Overloading কম্পাইল টাইম, Overriding রান টাইম।

HTML ও CSS-এর পার্থক্য কী?
HTML ওয়েব পেজের কাঠামো ও কনটেন্ট তৈরি করে, CSS ডিজাইন ও স্টাইলিং করে। HTML হলো হাড়, CSS হলো চামড়া।
Frontend ও Backend-এর পার্থক্য কী?
Frontend হলো ব্যবহারকারীর দেখা অংশ (HTML, CSS, JS)। Backend হলো সার্ভার-সাইড (PHP, Python, Node.js) যা ডাটাবেস অপারেশন করে।
HTTP ও HTTPS-এর পার্থক্য কী?
HTTPS হলো HTTP-এর সুরক্ষিত সংস্করণ। HTTPS SSL/TLS এনক্রিপশন ব্যবহার করে ডেটা সুরক্ষিত রাখে।

One-to-One (1:1)

Person ↔ Passport
Person (ব্যক্তি)
↔ (1:1)

উদাহরণ: একজন ব্যক্তির একটি মাত্র পাসপোর্ট, একটি পাসপোর্ট একজনের।

One-to-Many (1:N)

একটি এন্টিটির একটি ইনস্ট্যান্স অন্য এন্টিটির একাধিক ইনস্ট্যান্সের সাথে সম্পর্কিত।

Department → Students
Department (বিভাগ)
→ (1:N)
Student (ছাত্র) × অনেক

উদাহরণ: একটি বিভাগে অনেক ছাত্র, কিন্তু একজন ছাত্র একটি বিভাগে।

Many-to-Many (M:N)

একাধিক এন্টিটি একাধিক এন্টিটির সাথে সম্পর্কিত।

Students ↔ Courses
Student (ছাত্র) × অনেক
↔ (M:N)

উদাহরণ: একজন ছাত্র অনেক কোর্স নিতে পারে, একটি কোর্সে অনেক ছাত্র থাকতে পারে।

Weak Entity (দুর্বল এন্টিটি)

নিজস্ব Primary Key নেই, অন্য এন্টিটির উপর নির্ভরশীল।

4 SQL (Structured Query Language)

SQL কী?
SQL হলো ডাটাবেস ম্যানেজমেন্টের জন্য স্ট্যান্ডার্ড ভাষা। এটি দিয়ে ডেটা তৈরি, পড়া, আপডেট ও মুছে ফেলা (CRUD) যায়।

CREATE — টেবিল তৈরি

-- Department টেবিল তৈরি CREATE TABLE department ( dept_id INT PRIMARY KEY, dept_name VARCHAR(50) NOT NULL, building VARCHAR(30) ); -- Student টেবিল তৈরি CREATE TABLE student ( roll INT PRIMARY KEY, name VARCHAR(50) NOT NULL, age INT, dept_id INT, FOREIGN KEY (dept_id) REFERENCES department(dept_id) );

INSERT INTO department VALUES (1, 'CST', 'Block A'); INSERT INTO department VALUES (2, 'Civil', 'Block B'); INSERT INTO department VALUES (3, 'EEE', 'Block C'); INSERT INTO student VALUES (101, 'রহিম', 20, 1); INSERT INTO student VALUES (102, 'করিম', 21, 1); INSERT INTO student VALUES (103, 'সালমা', 19, 2); INSERT INTO student VALUES (104, 'ফাতিমা', 20, 1);

SUBQUERY — সাবকোয়েরি

-- সাবকোয়েরি: একটি কোয়েরির ভিতরে আরেকটি কোয়েরি -- যেসব ছাত্র CST বিভাগে SELECT * FROM student WHERE dept_id = ( SELECT dept_id FROM department WHERE dept_name = 'CST' ); -- সর্বোচ্চ বয়সী ছাত্র SELECT * FROM student WHERE age = (SELECT MAX(age) FROM student); -- IN সাবকোয়েরি SELECT * FROM student WHERE dept_id IN ( SELECT dept_id FROM department WHERE building = 'Block A' );

VIEW — ভিউ

-- VIEW: ভার্চুয়াল টেবিল CREATE VIEW student_details AS SELECT s.name, s.age, d.dept_name FROM student s JOIN department d ON s.dept_id = d.dept_id; -- VIEW ব্যবহার SELECT * FROM student_details; -- VIEW মুছে ফেলা DROP VIEW IF EXISTS student_details;

LEFT/RIGHT/FULL OUTER JOIN

-- LEFT JOIN: বাম টেবিলের সব রেকর্ড + ডান টেবিলের মিল SELECT s.name, d.dept_name FROM student s LEFT JOIN department d ON s.dept_id = d.dept_id; -- RIGHT JOIN: ডান টেবিলের সব রেকর্ড + বাম টেবিলের মিল SELECT s.name, d.dept_name FROM student s RIGHT JOIN department d ON s.dept_id = d.dept_id; -- FULL OUTER JOIN: দুই টেবিলের সব রেকর্ড (MySQL-এ UNION দিয়ে) SELECT s.name, d.dept_name FROM student s LEFT JOIN department d ON s.dept_id = d.dept_id UNION SELECT s.name, d.dept_name FROM student s RIGHT JOIN department d ON s.dept_id = d.dept_id;

INDEX — ইনডেক্স

-- INDEX: ডেটা দ্রুত খুঁজে পেতে CREATE INDEX idx_student_name ON student(name); -- Composite Index CREATE INDEX idx_student_dept_age ON student(dept_id, age); -- ইনডেক্স মুছে ফেলা DROP INDEX idx_student_name ON student;

5 ACID বৈশিষ্ট্য (Transaction Properties)

ACID কী?
ACID হলো ডাটাবেস ট্রানজেকশনের চারটি গুরুত্বপূর্ণ বৈশিষ্ট্য। এটি ডাটাবেসের নির্ভরযোগ্যতা ও সুসংগততা নিশ্চিত করে।
📊 ACID বৈশিষ্ট্য
A — Atomicity
C — Consistency
I — Isolation
D — Durability
বৈশিষ্ট্যবাংলা নামবিবরণউদাহরণ
A — Atomicity অবিভাজ্যতা একটি ট্রানজেকশনের সব অপারেশন সম্পন্ন হয় অথবা কোনটিই হয় না। আংশিক সম্পন্ন হওয়া যায় না। ব্যাংক থেকে টাকা তোলা — ব্যালেন্স কমে ও ক্যাশ বাড়ে — দুটোই হবে বা কোনটিই হবে না
C — Consistency সুসংগততা ট্রানজেকশনের আগে ও পরে ডাটাবেস সবসময় সুসংগত (consistent) অবস্থায় থাকে। সকল নিয়ম ও কনস্ট্রেইন্ট মানা হয়। অ্যাকাউন্ট A থেকে B-তে ১০০ টাকা পাঠালে — মোট ব্যালেন্স একই থাকে
I — Isolation বিচ্ছিন্নতা একাধিক ট্রানজেকশন একই সাথে চললেও একে অপরকে প্রভাবিত করে না। প্রতিটি ট্রানজেকশন মনে হয় যেন একা চলছে। দুজন একই সাথে একই অ্যাকাউন্টে টাকা জমা করলে — ভুল হয় না
D — Durability স্থায়িত্ব একবার ট্রানজেকশন সম্পন্ন হলে ডেটা স্থায়ীভাবে সংরক্ষিত হয়। সিস্টেম ক্র্যাশ হলেও ডেটা হারায় না। ব্যাংকে টাকা জমা হয়ে গেলে — সার্ভার বন্ধ হলেও ডেটা থাকবে
📌 ACID উদাহরণ: ব্যাংক ট্রান্সফার

ধরুন, অ্যাকাউন্ট A থেকে অ্যাকাউন্ট B-তে ১০০০ টাকা ট্রান্সফার করতে হবে:

Atomicity: A থেকে ১০০০ কমবে এবং B-তে ১০০০ বাড়বে — দুটোই হবে বা কোনটিই হবে না।

Consistency: ট্রান্সফারের আগে মোট ব্যালেন্স = ট্রান্সফারের পরে মোট ব্যালেন্স।

Isolation: অন্য কেউ একই সাথে A বা B-তে টাকা জমা করলে ভুল হবে না।

Durability: ট্রান্সফার সম্পন্ন হলে ডেটা ডাটাবেসে স্থায়ীভাবে সংরক্ষিত হবে।

-- ACID বৈশিষ্ট্য প্রদর্শনকারী SQL উদাহরণ -- ব্যাংক ট্রান্সফার: Account A থেকে B-তে 1000 টাকা START TRANSACTION; -- Step 1: A থেকে 1000 কমান UPDATE accounts SET balance = balance - 1000 WHERE account_id = 'A'; -- Step 2: B-তে 1000 বাড়ান UPDATE accounts SET balance = balance + 1000 WHERE account_id = 'B'; -- Step 3: সব ঠিক থাকলে COMMIT COMMIT; -- কোনো সমস্যা হলে ROLLBACK -- ROLLBACK;
📝 পরীক্ষার জন্য গুরুত্বপূর্ণ

ACID বৈশিষ্ট্য প্রায়ই পরীক্ষায় আসে। "ACID বলতে কী বোঝায়?" এবং "প্রতিটি বৈশিষ্ট্য ব্যাখ্যা করুন" — এমন প্রশ্ন আসতে পারে।

6 নর্মালাইজেশন (Normalization)

নর্মালাইজেশন কী?
নর্মালাইজেশন হলো ডাটাবেসে ডেটা রিডান্ডেন্সি (পুনরাবৃত্তি) কমানোর প্রক্রিয়া। এটি ডাটাবেসকে কার্যকর ও সংগঠিত করে।
ফর্মনামনিয়মবিবরণ
1NFপ্রথম স্বাভাবিক রূপপ্রতিটি সেলে একক মান থাকবেপুনরাবৃত্ত গ্রুপ থাকবে না
2NFদ্বিতীয় স্বাভাবিক রূপ1NF + পার্শিয়াল ডিপেন্ডেন্সি দূরNon-key অ্যাট্রিবিউট সম্পূর্ণভাবে PK-এর উপর নির্ভরশীল
3NFতৃতীয় স্বাভাবিক রূপ2NF + ট্রানজিটিভ ডিপেন্ডেন্সি দূরNon-key অ্যাট্রিবিউট অন্য non-key-এর উপর নির্ভরশীল নয়
📝 সহজ উদাহরণ

অনর্মালাইজড: একটি টেবিলে ছাত্রের নাম, রোল, বিভাগের নাম, বিভাগের বিল্ডিং — সব একসাথে।

নর্মালাইজড: Student টেবিলে শুধু ছাত্রের তথ্য + Department টেবিলে বিভাগের তথ্য। FK দিয়ে সংযুক্ত।

🎤 DBMS ইন্টারভিউ প্রশ্ন

DBMS ও RDBMS-এর পার্থক্য কী?
DBMS ডেটা সংরক্ষণ ও পরিচালনা করে। RDBMS হলো DBMS-এর একটি উন্নত রূপ যা টেবিল ভিত্তিক ডেটা স্টোর করে এবং রিলেশনশিপ তৈরি করে। MySQL, PostgreSQL RDBMS।
Primary Key ও Foreign Key-এর পার্থক্য কী?
Primary Key প্রতিটি রেকর্ড অনন্যভাবে শনাক্ত করে, NULL হতে পারে না। Foreign Key একটি টেবিলের কী যা অন্য টেবিলের PK-কে রেফার করে, NULL হতে পারে।
নর্মালাইজেশন কেন করা হয়?
ডেটা রিডান্ডেন্সি কমাতে, ডেটা অখণ্ডতা বজায় রাখতে, আপডেট/ইনসার্ট/ডিলিট অ্যানোমালি দূর করতে এবং ডাটাবেসকে কার্যকর করতে নর্মালাইজেশন করা হয়।

8085 ও 8086-এর পার্থক্য কী?
8085 হলো 8-বিট প্রসেসর, 16-বিট অ্যাড্রেস বাস, 64KB মেমরি। 8086 হলো 16-বিট প্রসেসর, 20-বিট অ্যাড্রেস বাস, 1MB মেমরি। 8086-এ পাইপলাইনিং ও সেগমেন্টেশন আছে।
Address Bus কী করে?
Address Bus মেমরি লোকেশনের ঠিকানা বহন করে। এটি CPU থেকে মেমরি বা I/O ডিভাইসে ঠিকানা পাঠায়। 16-বিট অ্যাড্রেস বাস 2^16 = 64KB মেমরি অ্যাড্রেস করতে পারে।
ALU কী করে?
ALU (Arithmetic Logic Unit) গাণিতিক অপারেশন (যোগ, বিয়োগ, গুণ, ভাগ) এবং লজিক্যাল অপারেশন (AND, OR, NOT, XOR) সম্পাদন করে।

OOP কেন গুরুত্বপূর্ণ?
OOP কোড পুনরায় ব্যবহারযোগ্য, রক্ষণাবেক্ষণ সহজ, ডেটা সুরক্ষিত রাখে এবং বড় প্রজেক্ট পরিচালনা সহজ করে। এছাড়া বাস্তব জগতের সমস্যা OOP দিয়ে সহজে মডেল করা যায়।
Python কেন শিখব?
Python সহজ, পড়তে সুন্দর, বিশাল লাইব্রেরি আছে। ওয়েব, AI/ML, ডেটা সায়েন্স, অটোমেশন — সব জায়গায় ব্যবহৃত হয়। চাকরির বাজারে Python ডেভেলপারের চাহিদা সবচেয়ে বেশি।
SQL কী?
SQL (Structured Query Language) হলো ডাটাবেস ম্যানেজমেন্টের স্ট্যান্ডার্ড ভাষা। এটি দিয়ে ডেটা তৈরি (CREATE), পড়া (SELECT), আপডেট (UPDATE) ও মুছে ফেলা (DELETE) যায়।
Primary Key কী?
Primary Key হলো একটি কলাম বা কলাম সমষ্টি যা প্রতিটি রেকর্ড অনন্যভাবে শনাক্ত করে। এটি NULL হতে পারে না এবং প্রতিটি টেবিলে একটিই থাকে।
8085 ও 8086-এর মধ্যে পার্থক্য কী?
8085 হলো 8-বিট, 16-বিট অ্যাড্রেস বাস, 64KB মেমরি। 8086 হলো 16-বিট, 20-বিট অ্যাড্রেস বাস, 1MB মেমরি। 8086-এ পাইপলাইনিং ও সেগমেন্টেশন আছে।
Encapsulation কী?
Encapsulation হলো ডেটা ও মেথডকে একসাথে বান্ডিল করা এবং বাইরের অ্যাক্সেস থেকে সুরক্ষিত রাখা। প্রাইভেট ভেরিয়েবল ও পাবলিক মেথড ব্যবহার করে করা হয়।
Inheritance কেন ব্যবহার করা হয়?
Inheritance কোড পুনরায় ব্যবহার করতে, কোড সংক্ষিপ্ত করতে এবং ক্লাসের মধ্যে সম্পর্ক স্থাপন করতে ব্যবহৃত হয়। চাইল্ড ক্লাস প্যারেন্টের বৈশিষ্ট্য ও মেথড পায়।
DBMS কী?
DBMS হলো ডাটাবেস তৈরি, পরিচালনা ও নিয়ন্ত্রণের সফটওয়্যার। MySQL, Oracle, PostgreSQL জনপ্রিয় DBMS। এটি ডেটা সুরক্ষা, ব্যাকআপ, কনকারেন্সি নিয়ন্ত্রণ করে।
HTML ও CSS কী?
HTML (HyperText Markup Language) ওয়েব পেজের কাঠামো তৈরি করে। CSS (Cascading Style Sheets) ডিজাইন ও স্টাইলিং করে। HTML হলো হাড়, CSS হলো চামড়া।

🔷 OOP ভাইভা প্রশ্ন

১. OOP বলতে কী বোঝায়?

OOP হলো অবজেক্ট ওরিয়েন্টেড প্রোগ্রামিং। এটি একটি প্রোগ্রামিং প্যারাডাইম যেখানে ক্লাস ও অবজেক্ট ব্যবহার করে প্রোগ্রাম লেখা হয়। OOP-এর চারটি মূল স্তম্ভ: Encapsulation, Inheritance, Polymorphism, Abstraction।

২. ক্লাস ও অবজেক্টের পার্থক্য বলুন।

ক্লাস হলো ব্লুপ্রিন্ট/টেমপ্লেট, অবজেক্ট হলো সেই টেমপ্লেটের নির্দিষ্ট উদাহরণ। ক্লাস একটি ধারণা, অবজেক্ট হলো প্রকৃত জিনিস। একটি ক্লাস থেকে অনেক অবজেক্ট তৈরি করা যায়।

৩. কনস্ট্রাক্টর কী?

কনস্ট্রাক্টর হলো একটি বিশেষ মেথড যা অবজেক্ট তৈরির সময় স্বয়ংক্রিয়ভাবে কল হয়। Python-এ __init__ মেথড। এটি অবজেক্টের প্রাথমিক মান সেট করে।

৪. এনক্যাপসুলেশন কী? উদাহরণ দিন।

এনক্যাপসুলেশন হলো ডেটা ও মেথডকে একসাথে বান্ডিল করা এবং সুরক্ষিত রাখা। উদাহরণ: ব্যাংক অ্যাকাউন্ট — ব্যালেন্স প্রাইভেট, deposit/withdraw পাবলিক মেথড।

৫. ইনহেরিট্যান্সের প্রকারভেদ বলুন।

সিঙ্গল (A→B), মাল্টিলেভেল (A→B→C), হায়ারার্কিক্যাল (A→B, A→C), মাল্টিপল (A,B→C), হাইব্রিড (মিশ্র)।

৬. পলিমরফিজম কী?

একই নামের মেথড বিভিন্নভাবে কাজ করে। Method Overloading (কম্পাইল টাইম) ও Method Overriding (রান টাইম) — দুই প্রকার।

৭. অ্যাবস্ট্রাকশন ও এনক্যাপসুলেশনের পার্থক্য কী?

অ্যাবস্ট্রাকশন = জটিলতা লুকানো (Implementation Hiding)। এনক্যাপসুলেশন = ডেটা সুরক্ষা (Data Hiding)। অ্যাবস্ট্রাকশন অ্যাবস্ট্রাক্ট ক্লাস দিয়ে, এনক্যাপসুলেশন প্রাইভেট মডিফায়ার দিয়ে করা হয়।

৮. Overloading ও Overriding-এর পার্থক্য বলুন।

Overloading: একই ক্লাসে একই নাম, আর্গুমেন্ট ভিন্ন (কম্পাইল টাইম)। Overriding: চাইল্ডে প্যারেন্টের মেথড পুনঃসংজ্ঞা (রান টাইম)।

৯. super() কী করে?

super() প্যারেন্ট ক্লাসের মেথড বা কনস্ট্রাক্টর কল করতে ব্যবহৃত হয়। চাইল্ড ক্লাসে প্যারেন্টের __init__ কল করতে ব্যবহার করা হয়।

১০. অ্যাবস্ট্রাক্ট ক্লাস কী?

অ্যাবস্ট্রাক্ট ক্লাস হলো এমন ক্লাস যার অবজেক্ট তৈরি করা যায় না। এতে অ্যাবস্ট্রাক্ট মেথড থাকে যা সাবক্লাসে অবশ্যই ইমপ্লিমেন্ট করতে হয়। Python-এ ABC মডিউল ব্যবহার করে তৈরি করা হয়।

🐍 পাইথন ভাইভা প্রশ্ন

১১. পাইথন কেন জনপ্রিয়?

সহজ সিনট্যাক্স, পড়তে সুন্দর, বিশাল লাইব্রেরি, ক্রস-প্ল্যাটফর্ম, OOP সমর্থন, AI/ML-এ ব্যবহৃত।

১২. Python-এ list ও tuple-এর পার্থক্য কী?

List = পরিবর্তনযোগ্য (mutable), [] ব্র্যাকেট। Tuple = অপরিবর্তনযোগ্য (immutable), () ব্র্যাকেট। Tuple দ্রুত।

১৩. Python-এ *args ও **kwargs কী?

*args = যেকোনো সংখ্যক পজিশনাল আর্গুমেন্ট নেয় (টাপল)। **kwargs = যেকোনো সংখ্যক কীওয়ার্ড আর্গুমেন্ট নেয় (ডিকশনারি)।

১৪. Python-এ Global ও Local ভেরিয়েবল কী?

Global = ফাংশনের বাইরে ডিক্লেয়ার, সব জায়গায় অ্যাক্সেসযোগ্য। Local = ফাংশনের ভিতরে ডিক্লেয়ার, শুধু ফাংশনের ভিতরে অ্যাক্সেসযোগ্য।

১৫. Python-এ Lambda ফাংশন কী?

Lambda হলো অনামিকা (anonymous) ফাংশন। একই লাইনে ফাংশন তৈরি করা যায়। যেমন: square = lambda x: x**2

১৬. Python-এ Decorator কী?

Decorator হলো এমন ফাংশন যা অন্য ফাংশনকে মডিফাই করে। @decorator_name সিনট্যাক্স ব্যবহার করে।

১৭. Python-এ Generator কী?

Generator হলো এমন ফাংশন যা yield কীওয়ার্ড ব্যবহার করে একটি একটি করে ভ্যালু রিটার্ন করে। মেমরি সাশ্রয়ী।

১৮. Python-এ Module ও Package কী?

Module = একটি .py ফাইল যতে ফাংশন/ক্লাস থাকে। Package = মডিউলের ফোল্ডার। import দিয়ে ব্যবহার করা হয়।

১৯. Python-এ Shallow Copy ও Deep Copy-এর পার্থক্য কী?

Shallow Copy = শুধু বাইরের অবজেক্ট কপি, ভিতরের রেফারেন্স একই। Deep Copy = সম্পূর্ণ কপি, কোনো রেফারেন্স নেই।

২০. Python-এ __str__ ও __repr__-এর পার্থক্য কী?

__str__ = ব্যবহারকারীর জন্য সুন্দর স্ট্রিং (print)। __repr__ = ডেভেলপারের জন্য অফিসিয়াল স্ট্রিং (repr)।

🌐 ওয়েব ভাইভা প্রশ্ন

২১. ওয়েবসাইট ও ওয়েব অ্যাপ্লিকেশনের পার্থক্য কী?

ওয়েবসাইট = তথ্য প্রদর্শনের জন্য (static)। ওয়েব অ্যাপ = ইন্টার‍্যাক্টিভ, ডেটা প্রসেস করে (dynamic)।

২২. Responsive Design কী?

রেসপন্সিভ ডিজাইন হলো এমন ডিজাইন যা যেকোনো স্ক্রিন সাইজে (মোবাইল, ট্যাবলেট, ডেস্কটপ) সুন্দরভাবে দেখায়। CSS Media Query ব্যবহার করে করা হয়।

২৩. Flexbox ও Grid-এর পার্থক্য কী?

Flexbox = 1D (এক ডাইমেনশনাল) লেআউট — সারি বা কলাম। Grid = 2D (দুই ডাইমেনশনাল) লেআউট — সারি ও কলাম দুটোই।

২৪. GET ও POST রিকোয়েস্টের পার্থক্য কী?

GET = ডেটা URL-এ দেখায়, বুকমার্ক করা যায়, সুরক্ষিত নয়। POST = ডেটা body-তে পাঠায়, বুকমার্ক করা যায় না, তুলনামূলক সুরক্ষিত।

২৫. API কী?

API (Application Programming Interface) হলো সফটওয়্যারের মধ্যে যোগাযোগের মাধ্যম। REST API সবচেয়ে জনপ্রিয়। JSON ফরম্যাটে ডেটা আদান-প্রদান করে।

🗄️ DBMS ভাইভা প্রশ্ন

২৬. ডাটাবেস কেন ব্যবহার করা হয়?

ডেটা সংগঠিত রাখতে, সহজে অ্যাক্সেস করতে, ডেটা সুরক্ষিত রাখতে, রিডান্ডেন্সি কমাতে এবং একাধিক ব্যবহারকারী একসাথে কাজ করতে পারে।

২৭. DDL ও DML-এর পার্থক্য কী?

DDL (Data Definition Language) = CREATE, ALTER, DROP — কাঠামো পরিবর্তন। DML (Data Manipulation Language) = SELECT, INSERT, UPDATE, DELETE — ডেটা পরিবর্তন।

২৮. ইনডেক্স (Index) কী?

ইনডেক্স হলো ডাটাবেসে ডেটা দ্রুত খুঁজে পাওয়ার জন্য একটি ডেটা স্ট্রাকচার। বইয়ের সূচিপত্রের মতো।

২৯. VIEW কী?

VIEW হলো ভার্চুয়াল টেবিল যা একটি SQL কোয়েরির ফলাফল দেখায়। ডেটা সংরক্ষণ করে না, শুধু কোয়েরি সংরক্ষণ করে।

৩০. Transaction কী?

Transaction হলো ডাটাবেস অপারেশনের একটি ইউনিট। ACID বৈশিষ্ট্য: Atomicity, Consistency, Isolation, Durability।

⚙️ মাইক্রোপ্রসেসর ভাইভা প্রশ্ন

৩১. মাইক্রোপ্রসেসর ও মাইক্রোকন্ট্রোলারের পার্থক্য কী?

মাইক্রোপ্রসেসর = শুধু CPU, বাইরের মেমরি ও I/O লাগে। মাইক্রোকন্ট্রোলার = CPU + মেমরি + I/O একটি চিপে।

৩২. Clock Speed কী?

ক্লক স্পিড হলো প্রসেসর প্রতি সেকেন্ডে কতবার কাজ করে তার হার। MHz/GHz-এ মাপা হয়।

৩৩. Flag Register কী?

Flag Register হলো একটি বিশেষ রেজিস্টার যা ALU-এর ফলাফলের স্ট্যাটাস দেখায়: Zero, Carry, Sign, Parity, Auxiliary Carry।

৩৪. Interrupt কী?

Interrupt হলো CPU-এর কাজ থামিয়ে অন্য কাজ করার সিগন্যাল। Hardware Interrupt (বাইরে থেকে) ও Software Interrupt (প্রোগ্রাম থেকে)।

৩৫. Memory Mapped I/O কী?

I/O ডিভাইসকে মেমরি অ্যাড্রেসে ম্যাপ করা। একই নির্দেশনা দিয়ে মেমরি ও I/O অ্যাক্সেস করা যায়।

🔷 আরও ভাইভা প্রশ্ন

৩৬-৫০. আরও OOP প্রশ্ন

৩৬. Interface কী? — অ্যাবস্ট্রাক্ট মেথডের সংগ্রহ। ক্লাস ইমপ্লিমেন্ট করতে হয়।

৩৭. Abstract Class ও Interface-এর পার্থক্য? — অ্যাবস্ট্রাক্ট ক্লাসে কনস্ট্রাক্টর ও ইনস্ট্যান্স ভেরিয়েবল থাকে, Interface-এ থাকে না।

৩৮. Coupling কী? — দুই ক্লাসের মধ্যে নির্ভরতা। কম কাপলিং ভালো।

৩৯. Cohesion কী? — একটি ক্লাসের সদস্যদের সম্পর্ক। বেশি কোহেশন ভালো।

৪০. Association, Aggregation, Composition কী? — ক্লাসের মধ্যে সম্পর্কের প্রকার।

৪১. Virtual Function কী? — রান টাইমে পলিমরফিজমের জন্য ব্যবহৃত ফাংশন (C++)।

৪২. this কীওয়ার্ড কী? — বর্তমান অবজেক্টকে নির্দেশ করে।

৪৩. Static মেথড কী? — অবজেক্ট ছাড়াই কল করা যায়। ক্লাস মেথড।

৪৪. Final কী? — পরিবর্তন করা যায় না (Java)। Final ক্লাসে ইনহেরিট করা যায় না।

৪৫. Garbage Collection কী? — অব্যবহৃত অবজেক্ট স্বয়ংক্রিয়ভাবে মুছে ফেলা।

৪৬. Immutable Object কী? — তৈরির পর পরিবর্তন করা যায় না। String (Java)।

৪৭. Singleton Pattern কী? — একটি ক্লাসের শুধু একটি অবজেক্ট তৈরি ডিজাইন প্যাটার্ন।

৪৮. MVC Pattern কী? — Model-View-Controller — সফটওয়্যার আর্কিটেকচার প্যাটার্ন।

৪৯. SOLID Principle কী? — ৫টি OOP ডিজাইন প্রিন্সিপল: S, O, L, I, D।

৫০. Design Pattern কী? — সফটওয়্যার ডিজাইনের পুনরায় ব্যবহারযোগ্য সমাধান।

৫১-৭০. আরও পাইথন প্রশ্ন

৫১. Python-এ GIL কী? — Global Interpreter Lock। একসময়ে একটি থ্রেড এক্সিকিউট করে।

৫২. Python-এ Memory Management কীভাবে হয়? — Garbage Collector ও Reference Counting ব্যবহার করে।

৫৩. List Comprehension কী? — একই লাইনে লিস্ট তৈরি: [x**2 for x in range(10)]

৫৪. Dictionary Comprehension কী? — {k: v for k, v in items}

৫৫. Map, Filter, Reduce কী? — ফাংশনাল প্রোগ্রামিং টুল।

৫৬. Python-এ Type Hinting কী? — ভেরিয়েবলের টাইপ নির্দেশ করা: def add(a: int, b: int) -> int

৫৭. Virtual Environment কী? — আলাদা Python এনভায়রনমেন্ট। pip install venv।

৫৮. pip কী? — Python Package Installer। থার্ড-পার্টি প্যাকেজ ইনস্টল করে।

৫৯. Python-এ @property কী? — Getter/Setter মেথড ডেকোরেটর দিয়ে তৈরি।

৬০. Python-এ Enum কী? — নির্দিষ্ট মানের সংগ্রহ। from enum import Enum।

৬১. Python-এ Dataclass কী? — @dataclass ডেকোরেটর দিয়ে সহজে ক্লাস তৈরি।

৬২. Python-এ Context Manager কী? — with স্টেটমেন্ট। রিসোর্স পরিচালনা।

৬৩. Python-এ Coroutine কী? — অ্যাসিঙ্ক্রোনাস ফাংশন। async/await।

৬৪. Python-এ Metaclass কী? — ক্লাস তৈরির ক্লাস। type() মেটাক্লাস।

৬৫. Python-এ Monkey Patching কী? — রান টাইমে ক্লাস/মডিউল পরিবর্তন।

৬৬. Python-এ Duck Typing কী? — "If it walks like a duck, it's a duck." টাইপ নয়, আচরণ গুরুত্বপূর্ণ।

৬৭. Python-এ MRO কী? — Method Resolution Order। ইনহেরিট্যান্সে মেথড খোঁজার ক্রম।

৬৮. Python-এ __slots__ কী? — মেমরি সাশ্রয়ী অ্যাট্রিবিউট ডিক্লেয়ারেশন।

৬৯. Python-এ Weakref কী? — দুর্বল রেফারেন্স। Garbage Collection প্রভাবিত করে না।

৭০. Python-এ Pickle কী? — অবজেক্ট সিরিয়ালাইজেশন। ফাইলে সংরক্ষণ।

৭১-৮৫. আরও ওয়েব ও DBMS প্রশ্ন

৭১. SEO কী? — Search Engine Optimization। ওয়েবসাইট সার্চ ইঞ্জিনে উচ্চতর র‍্যাঙ্ক।

৭২. Cookie ও Session-এর পার্থক্য কী? — Cookie ক্লায়েন্টে, Session সার্ভারে সংরক্ষিত।

৭৩. DNS কী? — Domain Name System। ডোমেন নামকে IP-তে রূপান্তর।

৭৪. SSL/TLS কী? — ওয়েব সুরক্ষা প্রোটোকল। HTTPS-এ ব্যবহৃত।

৭৫. CORS কী? — Cross-Origin Resource Sharing। ক্রস-ডোমেন রিকোয়েস্ট নিয়ন্ত্রণ।

৭৬. REST কী? — Representational State Transfer। API ডিজাইনের আর্কিটেকচার স্টাইল।

৭৭. MVC কী? — Model-View-Controller। ওয়েব ফ্রেমওয়ার্কের আর্কিটেকচার।

৭৮. ORM কী? — Object-Relational Mapping। ডাটাবেসকে অবজেক্ট হিসেবে ব্যবহার।

৭৯. Stored Procedure কী? — ডাটাবেসে সংরক্ষিত SQL কোড ব্লক।

৮০. Trigger কী? — ডাটাবেস ইভেন্টে স্বয়ংক্রিয়ভাবে এক্সিকিউট হয়।

৮১. Subquery কী? — একটি কোয়েরির ভিতরে আরেকটি কোয়েরি।

৮২. UNION কী? — দুই বা ততোধিক SELECT-এর ফলাফল একত্রিত করে।

৮৩. HAVING কী? — GROUP BY-এর পর শর্ত প্রয়োগ করে। WHERE গ্রুপের আগে, HAVING পরে।

৮৪. ACID কী? — Atomicity, Consistency, Isolation, Durability। Transaction-এর বৈশিষ্ট্য।

৮৫. Sharding কী? — ডাটাবেসকে ছোট ছোট অংশে ভাগ করা।

৮৬-১০০. আরও মাইক্রোপ্রসেসর প্রশ্ন

৮৬. Von Neumann Architecture কী? — প্রোগ্রাম ও ডেটা একই মেমরিতে থাকে।

৮৭. Harvard Architecture কী? — প্রোগ্রাম ও ডেটা আলাদা মেমরিতে থাকে।

৮৮. Pipeline কী? — একাধিক নির্দেশনা একসাথে প্রসেস করা।

৮৯. RISC ও CISC-এর পার্থক্য কী? — RISC = সহজ নির্দেশনা, দ্রুত। CISC = জটিল নির্দেশনা।

৯০. DMA কী? — Direct Memory Access। CPU ছাড়া সরাসরি মেমরি অ্যাক্সেস।

৯১. Cache Memory কী? — দ্রুত CPU মেমরি। L1, L2, L3।

৯২. ROM ও RAM-এর পার্থক্য কী? — RAM = পরিবর্তনযোগ্য, volatile। ROM = স্থায়ী, non-volatile।

৯৩. BCD কী? — Binary Coded Decimal। দশমিক সংখ্যাকে বাইনারিতে রূপান্তর।

৯৪. 2's Complement কী? — ঋণাত্মক সংখ্যা উপস্থাপনের পদ্ধতি।

৯৫. Stack কী? — LIFO ডেটা স্ট্রাকচার। PUSH/POP অপারেশন।

৯৬. Queue কী? — FIFO ডেটা স্ট্রাকচার।

৯৭. Bus Arbitration কী? — বাস ব্যবহারের অনুমতি নির্ধারণ।

৯৮. Handshaking কী? — ডেটা ট্রান্সফারের আগে যোগাযোগ প্রক্রিয়া।

৯৯. Baud Rate কী? — সিরিয়াল কমিউনিকেশনে ডেটা ট্রান্সফার রেট।

১০০. Microcontroller ও Microprocessor-এর পার্থক্য কী? — Microcontroller = CPU + RAM + ROM + I/O এক চিপে। Microprocessor = শুধু CPU।

🔷 OOP — দ্রুত রিভিশন

  • OOP = Object Oriented Programming — চার স্তম্ভ: Encapsulation, Inheritance, Polymorphism, Abstraction
  • ক্লাস = ব্লুপ্রিন্ট, অবজেক্ট = উদাহরণ
  • কনস্ট্রাক্টর = অবজেক্ট তৈরির সময় কল হয় (Python: __init__)
  • এনক্যাপসুলেশন = ডেটা সুরক্ষা (প্রাইভেট ভেরিয়েবল)
  • ইনহেরিট্যান্স = প্যারেন্টের বৈশিষ্ট্য চাইল্ডে পাওয়া
  • পলিমরফিজম = একই নাম বিভিন্ন কাজ (Overloading/Overriding)
  • অ্যাবস্ট্রাকশন = জটিলতা লুকানো (অ্যাবস্ট্রাক্ট ক্লাস)

🐍 পাইথন — দ্রুত রিভিশন

  • নির্মাতা: Guido van Rossum (1991) — ইন্টারপ্রেটেড, উচ্চ-স্তরের ভাষা
  • ডেটা টাইপ: int, float, str, bool, list, tuple, set, dict
  • লুপ: for, while — কন্ডিশনাল: if/elif/else
  • ফাংশন: def দিয়ে তৈরি — return দিয়ে ভ্যালু রিটার্ন
  • OOP: class, __init__, self, super()
  • ফাইল: open(), read(), write() — Exception: try/except/finally

🌐 ওয়েব — দ্রুত রিভিশন

  • HTML = কাঠামো, CSS = ডিজাইন, JavaScript = ইন্টার‍্যাক্টিভিটি
  • ফ্রন্টএন্ড = ব্রাউজারে রান করে, ব্যাকএন্ড = সার্ভারে রান করে
  • ক্লায়েন্ট → রিকোয়েস্ট → সার্ভার → ডাটাবেস → রেসপন্স → ব্রাউজার
  • PHP, Python (Django), Node.js = ব্যাকএন্ড প্রযুক্তি
  • Responsive Design = যেকোনো স্ক্রিনে সুন্দর দেখায়

🗄️ DBMS — দ্রুত রিভিশন

  • DBMS = ডাটাবেস ম্যানেজমেন্ট সিস্টেম, RDBMS = টেবিল ভিত্তিক
  • Primary Key = অনন্য শনাক্তকারী, Foreign Key = সম্পর্ক তৈরি
  • SQL: CREATE, INSERT, SELECT, UPDATE, DELETE
  • WHERE = শর্ত, ORDER BY = সাজানো, GROUP BY = গ্রুপ, JOIN = সংযুক্ত
  • নর্মালাইজেশন: 1NF (একক মান), 2NF (পার্শিয়াল ডিপেন্ডেন্সি), 3NF (ট্রানজিটিভ)

  • CPU = ALU + Control Unit + Register
  • Address Bus = ঠিকানা, Data Bus = ডেটা, Control Bus = নিয়ন্ত্রণ
  • ইনস্ট্রাকশন সাইকেল: Fetch → Decode → Execute → Store
  • রেজিস্টার: A, B, C, D, E, H, L, Flag, PC, SP
